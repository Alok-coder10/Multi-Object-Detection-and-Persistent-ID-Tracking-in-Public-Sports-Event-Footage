Technical Report

Multi-Object Detection and Persistent ID Tracking in Public Sports/Event Footage

April 2026 · YOLOv8m + ByteTrack · Video source ↗

1. Model / Detector Used

YOLOv8m (You Only Look Once, version 8 — medium variant) from the ultralytics library.

YOLOv8 is a single-stage anchor-free detector that predicts bounding boxes and class probabilities in a single forward pass. The medium variant was selected as the default because it achieves 50.2 mAP on COCO val2017, meaningfully outperforming the nano (37.3) and small (44.9) variants, while running at approximately 18–25 fps on a modern CPU.

Detection was restricted to COCO class 0 (person) via the classes=[0] parameter. The detection confidence threshold was set to 0.35 — conservative enough to reduce missed detections while filtering obvious noise.

2. Tracking Algorithm

ByteTrack-style custom tracker built on three components:

  • Kalman filter — constant-velocity motion model with 7-dimensional state vector: [cx, cy, area, aspect_ratio, vx, vy, v_area]
  • Hungarian algorithm — optimal bipartite assignment via the Munkres method (scipy linear_sum_assignment)
  • Two-stage association — the ByteTrack innovation: high-confidence detections first, low-confidence recovery second
Stage 1
High-confidence detections (conf ≥ 0.5) matched to active tracks using IoU cost matrix + Hungarian (threshold 0.30).
Stage 2
Low-confidence detections (0.1–0.5) matched against unmatched tracks from Stage 1 (threshold 0.20). Recovers occluded/blurred players.

3. Why This Combination

⚡
Speed vs accuracy
YOLOv8m balances both on CPU. Swap to YOLOv8l for GPU deployments.
◎
Crowd robustness
2-stage association recovers partially visible players that SORT discards entirely.
△
ID stability
Kalman filter predicts player position through occlusion — reduces ID switches.
⊡
No GPU for tracking
Unlike DeepSORT, ByteTrack uses pure IoU — no appearance embedding GPU needed.

4. How ID Consistency Is Maintained

ID consistency runs through a predict → match → update loop every frame:

  1. Predict: All active tracks Kalman-predict their next bounding box before seeing any detections. Every known track has an estimated position even if the detector misses it.
  2. Match: Hungarian algorithm finds globally optimal assignment between predicted boxes and detected boxes. A player detected slightly off their predicted position won't trigger a false ID switch.
  3. Buffer: Tracks that are unmatched remain alive for up to max_age=30 frames (~1 second). If the player re-appears within that window, the same ID is reused.
  4. Confirmation: New detections require min_hits=3 consecutive frames before becoming confirmed tracks. Suppresses flickering false-positive IDs.
  5. Monotonic counter: Track IDs assigned from a global integer counter. Once deleted, an ID is never reused — ensuring unique global count.

5. Challenges Faced

Occlusion in tight groups
Players clustering together cause overlapping bounding boxes. IoU matching degrades when two boxes are nearly identical. Kalman velocity helps but can't fully solve this without ReID features.
Similar appearance
Players in identical jerseys are visually indistinguishable. Pure IoU tracking has no appearance fallback when same-kit players cross paths after occlusion.
Fast camera motion
Pans and zooms invalidate all Kalman predictions simultaneously. The constant-velocity model assumes a stationary camera — fast pans briefly degrade all predictions.
Motion blur
Fast bowling shots or sprinting athletes produce blurred frames where YOLO confidence drops below the primary threshold. ByteTrack's Stage 2 recovery helps mitigate this.

6. Failure Cases Observed

  • Extended occlusion (>1 second): Occlusion longer than max_age frames forces ID reassignment on re-emergence. This is the most common failure mode.
  • Identical-looking subjects crossing: Two players in identical kits crossing paths may swap IDs because both Kalman prediction and IoU match are ambiguous.
  • Dense crowd scenes: With 20+ subjects in frame, the IoU matrix becomes large and individual match quality degrades near identical bounding box sizes.

7. Possible Improvements

Short-term
Appearance ReID
Integrate OSNet or FastReID to create visual feature embeddings per crop. Match on combined IoU + cosine distance for long-term occlusion recovery.
Short-term
Camera compensation
Lucas-Kanade optical flow to estimate global camera motion per frame and subtract from Kalman predictions before association.
Medium-term
Bird's-eye projection
Homography transform (calibrated from pitch markings) projects all player positions onto a 2D top-down field for tactical analysis.
Medium-term
Team clustering
K-means on player jersey colors in HSV space to automatically assign team labels. Works well in clean, well-lit footage.
Long-term
Transformer tracking
Explore MOTR or TrackFormer — end-to-end architectures that jointly model detection and tracking, eliminating manual IoU matching.
Long-term
Formal evaluation
Benchmark on MOT17/MOT20 using HOTA, IDF1, and MOTA metrics to formalize tracking quality beyond visual inspection.
